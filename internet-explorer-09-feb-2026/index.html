Client-Server Architecture

Client-server architecture is a system where a client (browser) sends a request to a server, and the server responds with the requested data.

Diagram:

Client (Browser)  ---- Request ---->  Server
Client (Browser)  <--- Response ----  Server

What Happens When You Open a Website

  1. You enter a URL in the browser (e.g., www.google.com).
  2. The browser sends a request to the server.
  3. DNS converts the domain name into an IP address.
  4. The server receives the request.
  5. The server processes the request and sends a response.
  6. The browser displays the webpage.

Conclusion

When a user enters a URL, the browser communicates with the server to retrieve data and display the webpage.
